# 仕様書：ベイズ更新ビジュアライザ

## 1. 教育上の狙い
第1章の最大の誤解——感度 $P(\text{警報}\mid F)$ と事後 $P(F\mid\text{警報})$ の混同（基準率の誤謬）——を、
数値と図で同時に壊す。低基準率では事後が直感より遥かに小さいことを体感させる。

## 2. 対応する誤解と解消法
| 誤解 | 解消法 |
|---|---|
| 感度99%なら鳴れば99%本物 | 事後メトリクスと内訳（偽陽性が多い）を並置 |
| P(A\|B)=P(B\|A) | 感度と事後を別メトリクスで対比 |
| 1回の検査で確定 | 逐次更新で複数警報の効果を可視化 |
| 感度を上げれば十分 | 特異度↑ と 感度↑ の事後改善幅を比較 |

## 3. 機能要件
- 入力：事前 P(F)、感度、誤警報率、母数 n、逐次警報回数。
- 出力：事後 P(F\|警報)・P(F\|無警報) メトリクス、n件内訳（TP/FP/FN/TN）棒、事後vs基準率曲線、逐次更新系列。
- 低基準率かつ高感度のとき警告で「基準率の誤謬」を明示。

## 4. 受け入れ基準
- [x] `core.py` self-test：P(F\|alarm)=0.1667、PPV==posterior、逐次2回≈0.798、基準率に単調、特異度改善>感度改善。
- [x] `AppTest`：既定値・基準率変更で例外なし。
- [x] `width="stretch"` 使用。

## 5. 数理（実装）
- `posterior_positive/negative`：ベイズの定理。
- `confusion_counts`：n件の TP/FP/FN/TN と PPV。
- `sequential_update/update_path`：オッズ×尤度比^k による独立更新。

## 6. 限界・拡張案
- 2状態（F/¬F）・2結果（警報/無）。多クラス・連続尤度は対象外。
- 拡張：連続検査値の尤度（ROC 連動）、共通原因（独立性破れ）の効果、事前分布のベイズ更新（ベータ-二項）。
