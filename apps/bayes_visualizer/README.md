# ベイズ更新ビジュアライザ（基準率の誤謬）

Module 1 の核心——**$P(\text{警報}\mid F)$（感度）と $P(F\mid\text{警報})$（事後）は別物**——を、
保護リレー（故障検知）を題材に対話的に体感するツールです。

関連ノート：[`notes/01_events_and_probability.md`](../../notes/01_events_and_probability.md) §6–7。

## 1. 何が体験できるか（学習目標）

1. **基準率の誤謬**：感度99%の優秀な警報でも、故障が稀（1%）なら事後は約17%。
2. **特異度の効き**：低基準率では、感度を上げるより**誤警報率を下げる**方が事後信頼度に効く。
3. **逐次更新**：独立な警報を重ねると事後が跳ね上がる（1回17%→2回80%）。「今日の事後＝明日の事前」。

## 2. 実行方法

```bash
streamlit run apps/bayes_visualizer/app.py
# 数理コアの自己テスト（Streamlit 不要）
python3 apps/bayes_visualizer/core.py
```
依存：`numpy`, `plotly`, `streamlit`。

## 3. 操作できるパラメータ

| パラメータ | 効果 |
|---|---|
| 事前確率 P(F)（基準率） | 下げると事後が急落（偽陽性の母数が効く） |
| 感度 P(警報\|F) | 上げても低基準率では事後はあまり改善しない |
| 誤警報率 P(警報\|¬F) | 下げる（特異度↑）と事後が大きく改善 |
| 内訳の母数 n | TP/FP/FN/TN の件数表示 |
| 逐次警報の回数 | 独立警報を重ねた事後の系列 |

## 4. 観察すべき点

- 既定（P(F)=0.01, 感度0.99, 誤警報0.05）で事後 **16.7%**。警報59.4件中 本物9.9件。
- 誤警報率を 0.05→0.01 にすると事後が 0.167→0.5 に（感度を1.0にしても0.168）。
- 逐次2回で約80%。ただし**警報が独立**（共通原因なし）が前提。

## 5. 数学的対応

$$P(F\mid\text{警報})=\frac{P(\text{警報}\mid F)P(F)}{P(\text{警報}\mid F)P(F)+P(\text{警報}\mid\neg F)P(\neg F)}$$
オッズ更新：事前オッズ $\times$ 尤度比 $\mathrm{LR}^+=P(\text{警報}\mid F)/P(\text{警報}\mid\neg F)$、独立 $k$ 回で $(\mathrm{LR}^+)^k$。

## 6. よくある誤解 / 電力への接続

- 「感度99%なら鳴れば99%本物」→ 基準率が低いと事後は大きく下がる。
- 「$P(A\mid B)=P(B\mid A)$」→ 一般に異なる（ベイズで反転）。
- **電力**：保護リレーの不要動作・警報の洪水。低基準率では誤警報率低減（特異度）が信頼度に効く。状態推定・異常検知にも直結。
