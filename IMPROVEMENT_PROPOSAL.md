# 改善提案 — UI・内容の精査（2026-06-29）

> ユーザーの声「**結局どうやって確率を使っていいのか、根本がわからない**」「人間UIとして分かりにくい」を
> 出発点に、教材（内容）と公開サイト（UI）を批判的に精査し、優先順位つきの改善案を出す。
> 自己弁護せず、**症状→原因→具体策**で書く。

---

## 1. 診断（ひとことで）

**網羅性は高いが「使えるようにする」設計が弱い。**
リファレンス（百科事典）としては良いが、初学者を「自力で確率を使える」状態へ運ぶ**一本道**が無い。

### 実測した根拠
| 観点 | 実測 | 含意 |
|---|---|---|
| 「確率の使い方／意思決定の手順」を主題にした文書 | **0本** | "結局どう使うか"の答えがどこにも無い＝ユーザーの不満の直接原因 |
| ノート→インタラクティブツールへの誘導リンク | **0** | 「読む」と「動かす」が分断。手を動かす導線が無い |
| ノート1本の行数 | **275〜376行**（計2480） | 密度が高い。初学者が要点に達する前に深掘りへ |
| 冒頭に短いまとめがあるノート | 8本中**3本**のみ | 全体像→詳細の順になっていない |
| 学習の「最短コース」明示 | 無し（ルート例はあるが埋もれる） | どこから始めるか分からない |

### 根本原因
- **構成が「概念の積み上げ順」**（事象→確率変数→分布→…）。"使い方"＝Module 6 まで**到達が遅い**。動機が続かない。
- **ツールが情報過多**（スライダー7＋プロット2＋メトリクス3…）で、**何を見れば良いか**の指示が無い＝「インタラクティブとして分からない」。
- 設計目標が**包括性**に寄り、ユーザーが欲しい**理解可能性（自力で使える）**とずれている。

---

## 2. 良い点（残すべき強み）
- 数理が正確（全数値を Python/cvxpy 検証）。誤解を潰す比較（密度≠確率 等）が秀逸。
- インタラクティブ5ツールが本番 Pages で動く（クリック読込で安定化済み）。
- 6形式を同一問題で比較する到達点が明確（x*=100〜137）。
- → **土台は強い。足りないのは「使い方への翻訳」と「導線」。**

---

## 3. 改善案（優先順位つき）

### 🥇 A1. 「確率の使い方」実践スパインを新設【最優先・根本対応】
- **問題**：概念は揃うが「曖昧な状況→具体的決定」を1本で示す文書が無い。
- **案**：1ページの実践ガイド `notes/how_to_use_probability.md`。
  1. **3ステップ**：①不確実な量に**名付ける**(ξ) → ②振れ方を**表す**(分布/シナリオ/集合) → ③**何を良しとするか**を選び**決める**(期待値/最悪/確率/CVaR)。
  2. **普遍的な完全例（パン屋：明日何個焼く?）**を最後までやり切る：品切れ損200/廃棄損50 → 臨界比0.8 → **113個焼く・品切れ確率20%**。平均100で焼くと期待損2500円、最適で大幅減。→ 同じ3ステップを**電力（需要・予備力）に1対1で写像**。
  3. **自分の問題の穴埋めテンプレート**（「私の不確実は__、__で表し、__を重視するから__形式、受け入れるリスクは__」）。
  4. **どのステップでどのノート/ツールを使うか**の地図。
- **効果：高**（ユーザーの「根本」に直接回答）。**工数：小〜中**。

### 🥈 B2. 「最短コース」を1本明示【On-ramp】
- **問題**：ナビが広大（ロードマップ4＋ノート8＋演習32＋ツール6）。開始点不明。
- **案**：ホーム冒頭とハブに **「まずこれだけ」3ステップ導線**：実践ガイド(A1) → Module 0（平均の罠を体感）→ コンパレータ（6形式の違いを体感）。所要30分。
- **効果：高**。**工数：小**。

### 🥉 B1. ツールに「ガイド付き観察」を入れる
- **問題**：ツールが操作盤のみで「何を見るか」の先生がいない＝分かりにくい。
- **案**：各ツール上部に **「👉 やってみよう」3ステップ**（例：PDF/CDF→「σを大きくして。山はどうなる? 面積は1のまま?」）。完了で次のヒント。
- **効果：高**。**工数：中**。

### A3. ノート↔ツールを織り込む
- **問題**：ノート→ツールのリンク0。
- **案**：各ノートの該当箇所に「**▶ ツールXを開いてYを動かして確かめよう**」を挿入。
- **効果：中〜高**。**工数：小**。

### A2. 各ノート冒頭に「30秒まとめ＋使う場面」
- **問題**：密度が高い。
- **案**：先頭に3行TL;DR＋「これは __ を決めたいときに使う」。
- **効果：中**。**工数：小**。

### B3. ツールの段階的開示
- **問題**：初見で全スライダーは過剰。
- **案**：基本スライダーのみ表示し、詳細は「詳細▼」に畳む。
- **効果：中**。**工数：中**。

---

## 4. 推奨する進め方
1. **A1（実践スパイン）** … 根本の疑問に答える。最優先で着手。
2. **B2（最短コース導線）** … 迷子をなくす。A1と同時。
3. **B1（ガイド付き観察）** … ツールを「分かる」体験に。
4. 余力で A3 / A2 / B3。

> いずれも既存資産は壊さず**上に薄く重ねる**改善。まず A1+B2 を作って「使い方の一本道」を通すのが、満足度への最短路。
