# 演習（発展）★★★ — Module 1

> 学習目標：独立性仮定の妥当性、希少事象、連続での「確率0」を、モデリング判断として扱う。

## A1-1 ★★★（20分）
**学習目標**：独立仮定 vs 共通原因故障。
3台の発電機が各々独立に確率 $p=0.05$ で故障する。
1. 「ちょうど $k$ 台故障」の確率（$k=0,1,2,3$）を二項分布で求めよ。
2. 「2台以上故障」の確率を求めよ。
3. ここに**共通原因故障**（確率 $0.02$ で3台が同時に停止、独立故障とは別口）が加わる。
   「2台以上故障」の確率はどう変わるか。独立仮定だけのモデルが**危険側に外す**理由を、$N{-}1$ 基準の文脈で論ぜよ。
4. 「独立と置いてよいか」を判断するために、現場で確認すべき情報を2つ挙げよ。

## A1-2 ★★★（15分）
**学習目標**：希少事象と条件付き確率の連鎖。
ある系統で「寒波」の確率は $0.1$。寒波時は需要急増の確率 $0.6$、平常時は $0.1$。
さらに需要急増時は供給支障の確率 $0.2$、非急増時は $0.01$。
1. 供給支障の確率 $P(\text{支障})$ を全確率の公式で求めよ。
2. 供給支障が起きたとき、それが寒波由来だった事後確率 $P(\text{寒波}\mid\text{支障})$ を求めよ。
3. この種の「条件の連鎖」をモデル化するとき、独立性をどこで仮定したか明示せよ。

## A1-3 ★★★（15分）
**学習目標**：連続での「確率0≠不可能」を意思決定に接続。
連続需要 $D$ について $P(D=100)=0$ である。
1. それでも「$D=100$ ちょうど」を狙って容量を 100 に決めるのが無意味な理由を、Module 0 の非対称コストと結びつけて述べよ。
2. 代わりに用いるべき量（区間確率・分位点・超過確率）を1つ選び、その量で容量を決める方針を1段落で提案せよ。
3. この方針が Module 6 のどの最適化形式（チャンス制約／CVaR 等）に対応するか予想せよ。
